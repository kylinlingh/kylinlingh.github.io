我的k8s社区之路

发表于 2021-01-13 | 更新于: 2021-01-16 | 分类于 k8s开发 | |

声明

k8s社区是一个人数非常庞大且流程完善的社区，我作为一个之前没有参加过其他github项目的新手，在刚开始加入社区时并没有找到很好的指导，所以才写了本文来记录自己是如何加入k8s社区的。本文的内容主要来源于两个渠道：

  • k8s的官方开发者指导
  • youtube上的new contributor workshop seattle 2018

注册CNCF

根据流程说明注册CNCF(云原生计算基金会)账号。

搭建测试环境

一旦涉及到代码变更，便需要先通过单元测试，整体测试和e2e测试，所以最好在本地搭建开发环境并通过了所有测试后才提交PR。

  • 拉取代码：
  • 在github上的k8s代码fork到自己的github账户上
  • 将刚刚fork的代码clone到本地的gopath/src/k8s.io/kubernetes目录下
  • 修改完代码后，在本地运行测试：
  • 执行所有的单元测试：在kubernetes目录下执行：make test
  • 只对某个特定的package运行单元测试： make test WHAT=./pkg/kubelet # run tests for pkg/kubelet
  • 对特定包及其所有子包运行单元测试：make test WHAT=./pkg/api/... # run tests for pkg/api and all its subpackages
  • 对多个子包运行单元测试：make test WHAT=./pkg/{kubelet,scheduler} # run tests for pkg/kubelet and pkg/scheduler

加入交流群

  • k8s社区首选slack进行交流，点击连接加入k8s的slack频道
  • 网上论坛
  • 加入邮件组
  • zoom线上会议,查看议程

Github工作流程

强烈推荐新手在k8s游乐场里按照上述的描述先体验整个github的工作流程。

三种主要的PR类型：

  • Bug Fix：kind/bug
  • Enhancement: kind/feature or kind/cleanup
  • KEP: Kubernets Enhancement Proposal

Approval Steps:

  • Submit
  • Add labels & notes
  • Bots say the PR is ok
  • SIGs/Owners/approve
  • Other contributors /lgtm
  • Tests pass
  • Tide merges the PR

加入特别兴趣小组(Sig: special interest group)

sig list

membership

每一个层级的会员会给你提的PR打上不同的标签，一般一个PR需要一个lgtm和一个approver才能merge

  • reviewer：lgtm(look good to me)
  • approver: approved

label

在首次参与k8s社区时，可以通过以下两个标签来寻找容易上手的issue：

# k8s
k8s学习笔记-四-k8s如何运行
